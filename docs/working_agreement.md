# GenesisPrediction v2
# Working Agreement（運用憲章）
Version: 1.0
Status: Active
Last Updated: 2026-02-19

---

## 0. 目的

本ドキュメントは、GenesisPrediction v2 の設計思想・運用原則・作業ルールを固定化するための憲章である。

目的は以下の3点：

1. 人間のミスを最小化する
2. AI（ChatGPT / Open-WebUI）の暴走を防ぐ
3. 将来の自分・協力者が迷わない構造を作る

本憲章はすべての作業より優先される。

---

# 1. 基本思想

## 1.1 SST（Single Source of Truth）

- 真実は `analysis/` 配下のみ
- GUIは `analysis/` のみを読む
- `data/` は素材であり、直接GUIは参照しない
- 手動編集による真実改変は禁止

analysis = 公開用の確定データ  
data = 原材料  
scripts = 生成装置

---

## 1.2 GUIの原則

- GUIは読み取り専用（Read Only）
- ロジックは持たない
- 計算は行わない
- 表示のみ担当する

表示の正しさは analysis 側の責任とする。

---

## 1.3 再現性

- 朝の儀式で同じ結果が再現できること
- ローカル差異を生まない
- 手動介入を最小化する

---

# 2. 作業ルール（絶対遵守）

## 2.1 1ターン1タスク

- 同時に複数の作業を行わない
- フェーズを混在させない
- 問題はスレッド分離する

---

## 2.2 差分パッチ禁止

- 必ず「完全ファイル全文」で提示
- 部分修正指示は禁止
- 手作業での編集を前提にしない

目的：人間のコピーミス防止

---

## 2.3 mainブランチは常に安定

- 実験は別ブランチ
- 不安定状態でpushしない
- タグで安定点を記録する

---

## 2.4 dataはGit管理しない

- `data/` は除外
- 真実は analysis にのみ出力
- 大容量ファイルはrepoに含めない

---

# 3. 朝の儀式原則

## 3.1 朝の儀式は聖域

- 毎日同じ手順
- 途中改造しない
- 完走が最優先

## 3.2 基本フロー

1. git pull
2. run_daily_with_publish.ps1
3. FX系スクリプト群
4. build_data_health.py
5. GUI確認
6. 必要時 deploy

---

# 4. 情報源更新ルール

## 4.1 更新タイミング

- フェーズ完了時のみ
- 思想変更時のみ
- 安定化後のみ

---

## 4.2 更新担当

原則：プロジェクトオーナーのみ

---

## 4.3 古い情報の扱い

- 上書きする
- 履歴はGitが保持する
- 文書内で過去仕様を残さない

---

# 5. AI連携ルール

## 5.1 ChatGPTの役割

- 設計
- 思想整理
- 構造判断
- フルファイル生成

## 5.2 Open-WebUIの役割

- 実務補助
- ローカル検証
- コード参照
- RAG検索

---

# 6. 禁止事項

- 手動でanalysisを書き換える
- GUIで計算処理を持つ
- 差分指示を受け入れる
- 複数作業を同時進行する
- 不安定状態でmainへpush

---

# 7. 最終原則

GenesisPredictionは：

「思想を持った再現可能システム」

である。

コードよりも構造を優先する。  
速度よりも安定を優先する。  
拡張よりも再現性を優先する。
