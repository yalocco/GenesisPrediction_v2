ファイル生成と運用の決め事（GenesisPrediction v2）
1. 本文書の位置づけ

本ドキュメントは、
GenesisPrediction v2 における すべてのチャット・設計相談・実装作業 に優先する
**運用契約（正本）**である。

この文書が参照・宣言されたチャットでは、
本ルールに反する挙動は不具合または違反として扱う。

2. 基本原則（最重要）
2.1 正本主義

正本は ファイルである

チャット上の説明は補助であり、正本ではない

正本は **共有ファイル倉庫（docs 等）**に置かれる

2.2 ファイル生成の原則

修正・追加は必ず以下のいずれかで行う：

✅ 完全なコード全文

✅ 完全なファイル生成（そのまま保存・置換可能）

以下は禁止する：

❌ 差分だけの提示

❌ 「ここを直して」形式

❌ 部分コードの貼り付け

❌ 手動編集前提の説明

3. 手動編集の禁止

ユーザーによる手動編集は禁止

編集が必要な場合は 必ず新しい完全ファイルとして再生成する

PowerShell here-string（@' ～ '@）の誤用は禁止

4. 責務分離の固定

以下は 不可逆に固定される：

Analyzer

データ生成・計算のみを行う

Digest / Sentiment / FX

生成物（JSON / CSV / PNG）を正本とする

GUI

正本を 読むだけ

再計算・推測・補完は禁止

5. ViewModel・生成物の扱い

ViewModel / CSV / PNG は正本

HTML / MD は閲覧・互換用途

欠損は欠損として扱う

404 や未生成は UI エラーにしない

6. チャット運用ルール

実行ログ・スクリーンショットは そのまま貼る

解釈・推測を加えない

設計相談と実装は分離する

本ルールに基づかない提案は採用しない

7. チャット開始時の宣言方法

新しいチャットでは、冒頭で以下を宣言する：

このチャットは
共有ファイル倉庫の
「ファイル生成と運用の決め事.md」
に従って運用する。

この宣言があった時点で、
本ルールは そのチャット全体に適用される。

8. 本文書の変更について

本文書の変更は 新バージョンとしてのみ行う

既存内容の暗黙変更は禁止

変更時は理由と影響範囲を明示する

9. 結論

このルールは、

人間とAIの協働を安定させ

チャットをまたいでも設計意図を失わせず

GenesisPrediction v2 を長期運用可能にする

ための 最低限かつ十分な契約である。

（GenesisPrediction v2 / File Generation & Operation Rules v1）
