# GenesisPrediction Pipeline Overview

Purpose  
This document describes the **complete system pipeline of GenesisPrediction**.

Its goal is to ensure that future developers, AI systems, and maintainers understand:

- How data flows through the system
- What each layer is responsible for
- Where calculations occur
- What the UI is allowed to do

This document acts as the **system map** of GenesisPrediction.

---

# Core Philosophy

GenesisPrediction follows a strict separation between:

```

Computation Layer
Presentation Layer

```

Computation happens in:

```

scripts/
analysis/

```

Presentation happens in:

```

app/static/

```

The UI **must never perform calculations**.

The UI **only displays artifacts generated by the analysis pipeline**.

---

# High-Level System Architecture

GenesisPrediction is structured into layered stages.

```

Observation
↓
Trend
↓
Signal
↓
Scenario
↓
Prediction

```

Each stage transforms information into a higher level of interpretation.

---

# 1. External Data Sources

GenesisPrediction ingests data from multiple sources.

Examples include:

```

News APIs
FX rate providers
Economic indicators
Geopolitical reports

```

Examples:

```

MediaStack
NewsAPI
AlphaVantage
ExchangeRate Host

```

These sources provide **raw external information**.

---

# 2. Fetcher Layer

Fetcher components retrieve data from external sources.

Location

```

scripts/

```

Examples

```

fetch_news.py
fetch_fx_rates.py

```

Responsibilities

- connect to external APIs
- download raw data
- normalize basic fields
- store results locally

Output format

```

analysis/raw/

```

Fetcher should **never interpret the data**.

Its only responsibility is **data retrieval**.

---

# 3. Observation Layer

Observation transforms raw data into structured observations.

Examples

```

sentiment analysis
news classification
event extraction

```

Location

```

analysis/world_politics/

```

Artifacts produced

```

sentiment_latest.json
daily_summary_latest.json
view_model_latest.json

```

Observation answers questions like:

- What happened today?
- What events occurred?
- What tone dominates the news?

Observation **does not predict the future**.

It only describes the present environment.

---

# 4. Trend Layer

Trend identifies **patterns over time**.

Examples

```

sentiment trend
market direction
volatility regime
historical comparisons

```

Trend answers:

> Are conditions improving, worsening, or stable?

Trend is the first layer where **temporal interpretation** appears.

---

# 5. Signal Layer

Signal detects **meaningful indicators** extracted from trends and observations.

Signals may include:

```

geopolitical escalation signals
financial instability signals
economic stress signals
policy change signals

```

Signal artifacts include:

```

signal_latest.json
early_warning_latest.json

```

Signals represent **structured warnings**.

Example signals:

```

energy shock
sanctions escalation
financial liquidity stress
military escalation risk

```

Signals are **inputs to scenario generation**.

---

# 6. Scenario Layer

Scenario engine generates possible future pathways.

Location

```

scripts/scenario_engine.py

```

Scenario branches typically include:

```

best_case
base_case
worst_case

```

Each scenario includes:

```

probability
confidence
drivers
watchpoints
assumptions
invalidation_conditions
supporting_signals

```

Scenario answers:

> What could happen next?

Scenario represents **structured future narratives**.

Artifacts

```

scenario_latest.json

```

---

# 7. Prediction Layer

Prediction summarizes the scenario landscape into an actionable forecast.

Prediction answers:

> What outcome is currently most likely?

Prediction may include:

```

risk level
dominant scenario
confidence score
forecast explanation

```

Artifacts

```

prediction_latest.json
prediction_history.json

```

Prediction is the **final interpretive layer**.

# Morning Ritual (Daily Execution Pipeline)

GenesisPrediction runs a daily execution process called the **Morning Ritual**.

Entry script

```

scripts/run_morning_ritual.ps1

```

Morning Ritual performs:

```

data fetch
observation update
signal generation
scenario generation
artifact refresh

```

This ensures the system updates automatically.

---

# Design Principles

GenesisPrediction follows several critical principles.

### Principle 1

Clear separation between **analysis and presentation**.

### Principle 2

Artifacts act as **stable contracts** between system layers.

### Principle 3

Each pipeline stage performs **one responsibility only**.

### Principle 4

Future maintainers must understand the system quickly.

Therefore documentation is considered **part of the architecture**.
