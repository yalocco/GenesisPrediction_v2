# Working Agreement (GenesisPrediction v2)

Version: 1.2
Last Updated: 2026-03-20

このドキュメントは「GenesisPrediction v2」を安全・確実に進めるための共同作業ルールです。
目的は **人間の事故（貼り間違え / 途中生成のコピペ / 手順抜け / 破壊的変更）を最小化** しつつ、
前進速度を落とさないことです。

---

# 1. 絶対ルール（破ると事故る）

## 1.1 1ターン = 1作業

* 1回のやり取りでは **1つの作業** だけ扱う
* 例：UI修正 / スクリプト修正 / Runbook更新 / デバッグ手順確定 など
* 併行して話し始めたら、必ず「どれを先にやるか」を決める

## 1.2 差分追加（部分修正・パッチ方式）の提案は禁止

* **必ず完全ファイル全文** で提示する
* ユーザー側は「ファイルを丸ごと置き換える」運用を基本とする
* 例外（緊急対応など）は必ず理由を明記してから

## 1.3 変更が出たら git diff → commit

* まず `git diff` を確認してから `git add` / `git commit` / `git push`
* 「試しに当ててみた」変更は、原則コミットしない

## 1.4 GUIは read-only（SST）

* GUI（`app/static/*.html`）は **JSON/CSV/PNG を読むだけ**
* GUI側で再計算・生成しない（再現性と事故防止）

# 2. コラボ手順（安全に進める）

## 2.1 ルート・パスの前提

* Repo Root: `D:\AI\Projects\GenesisPrediction_v2`
* GUI: `app/static/*.html`
* 分析成果（SST）: `data/world_politics/analysis/`
* digest: `data/digest/`（必要に応じて alias/コピー）

## 2.2 作業の基本フロー

1. 状況確認（ログ/スクショ/コマンド出力）
2. 変更対象ファイルを確定
3. **完全ファイル**を生成して上書き
4. `git diff` で意図通りか確認
5. `git add` → `git commit` → `git push`

## 2.3 “安全第一”の判断基準

* 迷ったら：

  * 破壊的変更を避ける
  * 既存安定版の動作を崩さない
  * まず観測（ログ/差分）を増やす
* “やっつけ”をすると後で必ず負債になる

## 2.4 dataはGit管理しない

* `data/` 配下は基本Git管理しない（サイズ・更新頻度・環境差）
* 例外（スキーマ/サンプル/小さな固定ファイル）は、必要性を説明してから

## 2.5 長大ファイルはダウンロードリンク方式（コピペ事故防止）

* 対象：概ね **300行以上** の HTML / CSS / JS / PowerShell / Python
* 目的：**貼り間違え / 途中生成 / 行欠け防止**
* 原則：

  * ChatGPTが完全ファイルを生成し、ダウンロードリンクを提示
  * ユーザーはダウンロードして上書き保存
  * コピペは原則禁止

# 3. 具体ルール（よくある事故を潰す）

## 3.1 既存ファイル確認ルール（新規追加）

AIはファイル生成・更新提案を行う前に、必ず以下を確認すること：

### 必須

* 対象ファイルが既に存在するか確認する
* 存在する場合は **ユーザーに提出を求める**
* 内容を確認した上で、それをベースに作業する

### 禁止事項

* 既存ファイルを確認せずに新規生成すること
* 推測で既存構造を再現すること
* 既存実装を無視して上書き提案すること

### 例外

* ユーザーが「新規作成」と明示した場合
* リポジトリ上で未作成が確定している場合

---

## 3.2 “日付（as_of）”の扱い

* 表示がおかしい場合はまず **source JSON を疑う**
* UIで無理やり修正しない

## 3.3 画像フォールバック

* 403/404は通常
* 安全に代替表示する

## 3.4 新スレ基準

* ログが長くなったら新スレ
* 「目的 / 現状 / 次の一手」を明示

---



## 3.6 完全ファイルはダウンロード方式で提供（強化）

### 必須（MUST）

* 完全ファイルは **必ずダウンロードリンクで提供する**
* インライン長文コードの出力は禁止（特にスレ終盤・高負荷時）
* 対象：HTML / CSS / JS / Python / PowerShell / JSON / 300行未満でも可

### 理由

* 生成速度の安定化
* 途中欠落・短文化の防止
* コピペ事故の防止
* 再現性の確保

### 手順

1. ChatGPTが完全ファイルを生成
2. ダウンロードリンクを提示
3. ユーザーはダウンロードして上書き保存
4. `git diff` で確認

### 例外

* 数十行の小さなスニペットのみ（ユーザーが明示的に要求した場合）
