# Chat Operating Rules (GenesisPrediction v2)
Version: 1.0
Status: Active
Last Updated: 2026-03-05

## 0. 目的
AI共同開発を “事故なく高速に” 継続するための、チャット上の作業規約（短縮版）。
公式の詳細は docs/working_agreement.md を正とする。

## 1. 絶対ルール
1) 1ターン = 1作業（複数案件を混ぜない）
2) 差分追加（部分修正・パッチ方式）の提案は禁止
3) 出力は **完全ファイル全文**（置き換え前提）
4) 変更が出たら git diff → commit（試し当てを混ぜない）
5) GUI（app/static）は read-only（SSTを読むだけ）

## 2. 受け渡しルール（コピペ事故防止）
- 200〜300行超のファイルは、原則「ダウンロード運用」で受け渡す
- 例外は最小（小さな1〜2行相談など、明確に安全な場合のみ）

## 3. スレ運用（重くなったら）
- スクショ/ログ/ファイルが増えて重くなったら新スレ
- 新スレ1投稿目に貼るべき項目:
  - 目的 / 現在の状態 / 確定情報 / 次の一手 / ルール（この規約）

## 4. “何を直すべき？”の優先順位
- 表示がおかしい → まず SST（analysis）が正しいか確認
- SSTが正しい → GUI（app/static）問題
- SSTが古い/欠ける → scripts / pipeline 問題
